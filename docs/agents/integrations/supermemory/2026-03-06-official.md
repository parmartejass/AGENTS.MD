# Supermemory Integration Notes (Retrieved 2026-03-06, re-verified 2026-03-08)

## Sources

- https://supermemory.ai/docs/supermemory-mcp/setup
- https://supermemory.ai/docs/supermemory-mcp/mcp
- https://github.com/supermemoryai/supermemory-mcp

## Repo-relevant facts

- Supermemory MCP Server 4.0 documents the stable remote MCP URL `https://mcp.supermemory.ai/mcp` for manual client configuration.
- The hosted server uses OAuth by default, so repo-owned config can stay non-secret and let each client handle user authentication interactively.
- Repo-managed config makes the server discoverable, but first use still requires the user to complete the client-side OAuth flow locally.
- API key authentication and project-scoping headers are supported, but those headers are optional and user-specific, so they should stay out of repo-owned shared config.
- The public GitHub repo warns that the older MCP v1 flow is being deprecated, so repo docs should not keep pointing at unique legacy `/sse` URLs as the primary path.

## Repo decision

- Connect shared MCP JSON clients through `docs/agents/mcp/shared/mcp.json`.
- Connect Codex through the shared project config at `docs/agents/settings/codex/config.toml`.
- Keep OAuth as the default shared path; do not store API keys, bearer headers, or project-specific headers in repo-owned config.
