# Nia Integration Notes (Retrieved 2026-03-06, re-verified 2026-03-08)

## Sources

- https://docs.trynia.ai
- https://trynia.ai

## Repo-relevant facts

- Nia is a context-retrieval and indexing system that can search repositories, docs, papers, datasets, local folders, and Slack workspaces.
- Nia's official integration docs still recommend the remote MCP path for Claude Code, Cursor, and Codex.
- Nia's official agent-rules docs still publish preconfigured runtime files for Claude, Cursor, and Codex as user/runtime artifacts, separate from this repo's canonical skill bundle.
- The repo-owned Nia asset should remain a non-secret skill bundle that reads credentials from environment variables or local secret files.
- Repo .env files can help local shells, but they are not the canonical cross-device auth bridge for the bundled shell wrappers.
- Nia belongs in the skill/integration layer, not in mixed platform config files.

## Repo decision

- Keep the canonical Nia integration under `docs/agents/skills/nia/`.
- Keep secrets out of repo-owned files.
- Use platform adapters only to expose the canonical bundle in each supported runtime.
